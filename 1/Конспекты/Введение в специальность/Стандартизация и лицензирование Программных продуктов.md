# Стандартизация программных продуктов
Стандартизация - деятельность по установлению правил и характеристик в целях их добровольного многократного использования направленная на достижение упорядоченности и конкурентно-способности продукции.

В области ПО стандартизация решает следующие задачи: 
- Повышение качества программных средств
- Управление жизненным циклом ПО
- обеспечение совместимости и переносимости
- Документация и регламентация - разработка и оформление технической и пользовательской документации в соответствии с установленными нормами

# Лицензирование
Процесс предоставления права использования ПО на определенных условиях. ПО является объектом индивидуальной собственности и его использование регулируется авторским правом. А лицензия определяет объем прав

### Виды лицензий на ПО

1. Проприетарное (Коммерческие) лицензии - лицензия по которым издатель даёт разрешение получателю использовать одну или несколько копий программы, но при этом сам остаётся правообладателем всех этих копий. (Microsoft Windows)
2. Свободная/открытая модель лицензирования (Free/OpenSource software, FOSS) - свободное ПО с открытым исходным кодом. Такая модель лицензирования предполагает, что пользователи могут свободно использовать, копировать, изучать и изменять исходный код (Firefox)
3. Условно-бесплатная (Shareware) - ПО которое распространяется бесплатно, но с определёнными ограничениями. Обычно за дальнейшую поддержку пользователю предлагают заплатить. (WinRar)
4. Бесплатная (Freeware) - бесплатное ПО, которое доступно для использования без денежных отчислений. Как правило, в лицензии таких программ указано, что с ними можно делать всё, что угодно, кроме изменения кода. (OpenOffice)
